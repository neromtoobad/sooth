## 2. Module anatomy

Every contract is a module. Attribute goes on **both** the struct and the impl block.

```rust
use odra::prelude::*;   // brings in Var, Mapping, SubModule, Address, etc.

#[odra::module]
pub struct Flipper {
    value: Var<bool>,
}

#[odra::module]
impl Flipper {
    pub fn init(&mut self) {        // optional one-time constructor; name MUST be `init`
        self.value.set(false);
    }

    pub fn set(&mut self, value: bool) {
        self.value.set(value);
    }

    pub fn flip(&mut self) {
        self.value.set(!self.get());
    }
}
```

- `pub fn` in the `#[odra::module]` impl → contract entry point. Non-pub fns stay internal.
- `&mut self` = state-mutating entry point; `&self` = read-only.

### Storage types

```rust
#[odra::module]
pub struct DogContract {
    barks: Var<bool>,
    weight: Var<u32>,
    name: Var<String>,
    walks: Var<Vec<u32>>,              // works, but Var serializes whole value each r/w
}

#[odra::module]
pub struct DogContract2 {
    name: Var<String>,
    friends: Mapping<String, u32>,     // no iteration; cheap point reads/writes
}

#[odra::module]
pub struct DogContract3 {
    name: Var<String>,
    walks: List<u32>,                  // iterable; push(v), len(), iter()
}
```

APIs:
- `Var<T>`: `set(value)`, `get() -> Option<T>`, `get_or_default()`, `get_or_revert_with(Error)`
- `Mapping<K,V>`: `set(&key, value)`, `get_or_default(&key)` (also `get(&key) -> Option<V>`)
- `List<T>`: `push(value)`, `len()`, `iter()`

### Custom types

```rust
#[odra::odra_type]
pub struct Dog {
    pub name: String,
    pub age: u8,
    pub owner: Option<Address>
}
```

### SubModule composition

```rust
#[odra::module]
pub struct ModulesContract {
    pub math_engine: SubModule<MathEngine>
}
```
SubModule creates its own keyspace; call its public methods as `self.math_engine.foo()`.

Expose submodule methods as entry points with `delegate!` inside the `#[odra::module]` impl:

```rust
delegate! {
    to self.module_name {
        fn function_name(&mut self, param: Type) -> ReturnType;
        fn another_function(&self) -> ReturnType;
    }
    to self.another_module {
        fn delegated_function(&mut self, param: Type);
    }
}
```

### Events

```rust
#[odra::event]
pub struct PartyStarted {
    pub caller: Address,
    pub block_time: u64,
}

// emit inside an entry point (CES standard):
self.env().emit_event(PartyStarted {
    caller: self.env().caller(),
    block_time: self.env().get_block_time()
});

// Casper 2.0+ native events also exist:
self.env().emit_native_event(NativePartyStarted { ... });
```

Register events on the module so they land in the contract schema (recursive — submodules'
registered events are picked up automatically):

```rust
#[odra::module(events = [PartyStarted, NativePartyStarted])]
pub struct PartyContract;
```

### Errors & reverts

```rust
#[odra::odra_error]
pub enum Error {
    OwnerNotSet = 1,
    NotAnOwner = 2
}

#[odra::module(errors = Error)]
pub struct OwnedContract { /* ... */ }

// throw:
self.env().revert(Error::NotAnOwner)
// or on Option-like storage reads:
self.owner.get_or_revert_with(Error::OwnerNotSet)
```

Rules: field-less enums only, explicit unique discriminants, **unique across the whole project**.

### ContractEnv essentials (inside contract code)

`self.env().caller()`, `self.env().get_block_time()`, `self.env().emit_event(..)`,
`self.env().revert(..)`, `self.env().self_address()` `// derived: self_address from prelude`.

## 3. CEP-18 token (sUSD)

Exact path (verified): **`odra_modules::cep18_token::Cep18`** — requires the `odra-modules` crate.

```rust
use odra_modules::cep18_token::Cep18;

#[odra::module]
pub struct OurToken {
    token: SubModule<Cep18>,
}

#[odra::module]
impl OurToken {
    pub fn init(&mut self, name: String, symbol: String, decimals: u8, initial_supply: U256) {
        // NOTE the arg order Cep18 expects: symbol, name, decimals, initial_supply
        self.token.init(symbol, name, decimals, initial_supply);
    }
}
```

- Standard entry points (`transfer`, `transfer_from`, `approve`, `balance_of`, `allowance`,
  `total_supply`, …) are exposed via `delegate! { to self.token { ... } }`.
- **Privileged internal ops** (no access control — gate them yourself in the wrapper):
  - `self.token.raw_mint(&account, &amount);`
  - `self.token.raw_burn(&account, &amount);`
  - `self.token.raw_transfer(&from, &to, &amount);`  // contract-controlled transfer, ignores allowance
- For SOOTH sUSD: expose a `pub fn mint(&mut self, to: Address, amount: U256)` that checks
  `self.env().caller()` against an admin `Var<Address>` then calls `raw_mint`. `// derived`
- Faster start: `cargo odra new --name ourcoin --template cep18`.

## 4. Factory pattern & cross-contract calls (MarketFactory → BinaryMarket)

### Built-in factory codegen

Put `factory=on` on the **child** contract's struct AND impl — Odra generates a
`{{Name}}Factory` contract for you:

```rust
#[odra::module(factory=on)]
pub struct Counter {
    value: Var<u32>
}

#[odra::module(factory=on)]
impl Counter {
    pub fn init(&mut self, value: u32) {
        self.value.set(value);
    }
}
```

Generated factory entry points:
- `new_contract(contract_name: String, ...init args...) -> (Address, URef)` — deploys a child instance
- `upgrade_child_contract(contract_name: String)`
- `batch_upgrade_child_contract<T: Into<odra::casper_types::RuntimeArgs>>(args: BTreeMap<String, T>)`

Test usage (note `CounterHostRef::new(address, env)` to attach a host ref to a returned address):

```rust
#[test]
fn test_factory() {
    let env = odra_test::env();
    let mut factory_ref = CounterFactory::deploy(&env, NoArgs);
    let (address, _access_uref) = factory_ref.new_contract(String::from("Counter"), 10);
    let mut counter_ref = CounterHostRef::new(address, env);
    counter_ref.increment();
    assert_eq!(counter_ref.value(), 11);
}
```

### Cross-contract calls (contract → contract)

Every `#[odra::module]` gets a generated **`{{ModuleName}}ContractRef`** (implements all public
entry points + `address()`). Call another deployed contract by address:

```rust
MathEngineContractRef::new(self.env(), math_engine_address).add(3, 5)
```

Or hold the ref in storage with `External`:

```rust
pub struct CrossContract {
    pub math_engine: External<MathEngineContractRef>
}

pub fn add_using_another(&self) -> u32 {
    self.math_engine.add(3, 5)
}
```

For third-party contracts you don't have source for, declare the ABI as a trait:

```rust
#[odra::external_contract]
pub trait Adder {
    fn add(&self, n1: u32, n2: u32) -> u32;
}
// generates AdderContractRef; use AdderContractRef::new(self.env(), address).add(3, 5)
```

SOOTH note: BinaryMarket holds the sUSD address and moves funds via a Cep18-shaped
`ContractRef` (e.g. `OurTokenContractRef::new(self.env(), self.susd.get().unwrap()).transfer_from(...)`). `// derived`

## 5. Testing (odra-test)

Plain `#[test]` fns; get a host env from `odra_test::env()`.

```rust
use odra::{host::Deployer, prelude::*};

#[test]
fn env() {
    let test_env: HostEnv = odra_test::env();
    test_env.set_caller(test_env.get_account(0));
    let init_args = TestingContractInitArgs {
        name: "MyContract".to_string()
    };
    let testing_contract = TestingContract::deploy(&test_env, init_args);
}
```

- Deploy: `MyModule::deploy(&env, MyModuleInitArgs { ... })` → returns a **HostRef** with all entry points.
- Init args: Odra auto-generates `{{ModuleName}}InitArgs` when `init` takes args; use
  `odra::host::NoArgs` when it doesn't (or there's no `init`). Both implement `InitArgs`.

HostEnv API:

| fn | purpose |
|---|---|
| `set_caller(&self, address: Address)` | switch caller for subsequent calls/deploys |
| `get_account(&self, n: usize) -> Address` | pre-funded test accounts (0, 1, 2, …) |
| `balance_of<T>(&self, addr: &T) -> U512` | **native CSPR** balance (U512) — CEP-18 balances come from the token's own `balance_of` entry point |
| `block_time(&self) -> u64` / `block_time_secs(&self)` | current block time (ms / s) |
| `advance_block_time(&self, time_diff: u64)` | move time forward (ms) — use for market close/resolution tests |
| `emitted_event<T, R>(&self, contract_address: &R, event: T) -> bool` | exact event assertion |
| `emitted(&self, contract, "EventName")` | name-only event assertion |
| `events_count(&self, contract)` | number of events emitted |

```rust
assert!(test_env.emitted_event(&party_contract, PartyStarted { /* exact fields */ }));
assert_eq!(test_env.events_count(&party_contract), 1);
```

Reverts: every entry point also gets a `try_{{entry_point}}` variant returning `OdraResult`:

```rust
assert_eq!(
    owned_contract
        .try_change_name("NewName".to_string())
        .unwrap_err(),
    Error::NotAnOwner.into()
);
```

(Calling the non-`try_` version on an error panics the test.)

## 6. Livenet backend (deploy to Casper testnet)

Env vars (exact names, verified):

```
ODRA_CASPER_LIVENET_SECRET_KEY_PATH=<path to secret_key.pem>
ODRA_CASPER_LIVENET_NODE_ADDRESS=<node address>
ODRA_CASPER_LIVENET_EVENTS_URL=<events url>
ODRA_CASPER_LIVENET_CHAIN_NAME=casper-test        # "casper" = mainnet
# optional:
ODRA_CASPER_LIVENET_KEY_1=<additional account key path>
ODRA_CASPER_LIVENET_TTL=<deploy ttl, default 5 min>
CSPR_CLOUD_AUTH_TOKEN=<token>                     # when using CSPR.cloud endpoints
```

Deploy scripts are feature-gated binaries:

```toml
[bin]
name = "erc20_on_livenet"
path = "src/bin/erc20_on_livenet.rs"
required-features = ["livenet"]
test = false
```

```bash
cargo run --bin erc20_on_livenet --features=livenet
# multiple env sets: integration.env / mainnet.env loaded over base .env
ODRA_CASPER_LIVENET_ENV=integration cargo run --bin erc20_on_livenet --features=livenet
```

In the livenet script (`env` here is the livenet HostEnv, e.g. `odra_casper_livenet_env::env()` `// derived`):

```rust
env.set_gas(450_000_000_000u64);          // set BEFORE every deploy/call that costs gas
let token = Erc20::deploy(env, init_args);  // deploy
let token = Erc20::load(env, address);      // attach to an already-deployed contract
```

`set_gas` is in motes (450_000_000_000 = 450 CSPR — deploys are expensive; plain calls need far less).

## 7. Math / numeric types

- Big ints come from casper-types re-exported by odra: **`odra::casper_types::U256`** and
  **`odra::casper_types::U512`** (`U256` is what Cep18 uses everywhere; native balances are `U512`).
- `RuntimeArgs` also lives there: `odra::casper_types::RuntimeArgs` (seen in factory codegen).
- `Address` comes from `odra::prelude::*` (Odra's unified account/contract address type).
- No floats on-chain. CPMM fixed-point: do math in U256 with a `1e18`-style scale factor,
  multiply before divide, and watch for overflow — U256 * U256 can exceed U256; U512 is available
  for wide intermediates. `// derived guidance`

## SOOTH quick map

- `sUSD` = wrapper over `SubModule<Cep18>` + admin-gated `mint` calling `raw_mint`.
- `BinaryMarket` = own module; holds sUSD `Address`; moves collateral via generated
  `...ContractRef::new(self.env(), addr)`; emits `#[odra::event]`s on every trade for the dashboard.
- `MarketFactory` = either `#[odra::module(factory=on)]` on BinaryMarket (free codegen) or a
  hand-rolled registry module that stores child addresses in a `List<Address>`.
- Tests: `odra_test::env()`, `advance_block_time` to pass market close, `try_*` for revert paths.
- Deploy: livenet feature binary, `ODRA_CASPER_LIVENET_CHAIN_NAME=casper-test`, `env.set_gas` first.
